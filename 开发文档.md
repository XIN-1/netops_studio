# NetOps Studio 开发文档（规划版 v0.2）

> 面向网络工程师 / IT 运维的集成化桌面运维工作台。覆盖日常运维全场景，桌面端优先（Windows / Linux / macOS），Web 暂不做。

---

## 1. 项目概述

| 项 | 内容 |
|----|------|
| 名称 | NetOps Studio / 网维工作台 |
| 定位 | 集成化桌面运维工作台，顶替分散的 ping / 扫描 / 测速 / 配置 / 监控 / 排障小工具 |
| 目标 | 查阅快、数据全、可扩展；一套工具覆盖日常运维绝大多数场景 |
| 范围 | **桌面端优先**（Win/Linux/macOS）；Web 端暂缓 |
| 形态 | 本地桌面应用（PySide6），离线可用，可选接入远程采集节点 |
| 与旧版关系 | 参考现有「网维助手Pro」功能，**全部新写、不复用其代码**；补齐外网测速、多厂商 IP 冲突、拓扑、监控告警等旧版缺失项 |

**核心设计原则**
- `core/` 引擎层与 UI 完全解耦（纯 Python，不依赖 Qt），便于单测与未来 Web 复用。
- 插件式 Tab 注册（`TabRegistry`），新功能即"注册一个新 tab + 复用 core"。
- 协作式异步（`AsyncWorker`），禁用线程强杀（`Thread.terminate`）。
- 事件总线（`EventBus`）解耦模块与仪表盘。
- 依赖注入（`Container`）管理服务单例。

## 3. 系统架构（四层）

```
┌──────────────────────────────────────────────────────────────┐
│  表现层 (GUI)            PySide6 各模块 Widget，经 TabRegistry 懒加载  │
├──────────────────────────────────────────────────────────────┤
│  应用服务层              TabRegistry / EventBus / AsyncWorker       │
│                         Theme + i18n / Container(DI)               │
├──────────────────────────────────────────────────────────────┤
│  核心引擎层 (core)       纯 Python 逻辑，**不依赖 Qt**，可单测、可复用  │
│                         discovery / diagnostics / speedtest / subnet │
│                         ipam / config_mgmt / monitor / ...          │
├──────────────────────────────────────────────────────────────┤
│  基础设施与数据层         Config / DeviceStore / History / Crypto     │
│                         Logging + 跨平台适配（OS 命令分支、iperf3 分发）│
└──────────────────────────────────────────────────────────────┘
                        运行时：Windows / Linux / macOS + Python 3.11+
```

**依赖方向**：GUI → 应用服务 → core → 基础设施。禁止反向依赖（core 不得 import PySide6）。

## 6. 核心模块设计（MVP）

### 6.1 子网计算器 `core/subnet.py`
```python
import ipaddress
from dataclasses import dataclass, field

@dataclass
class SubnetResult:
    network: str
    broadcast: str
    netmask: str
    wildcard: str
    prefixlen: int
    host_count: int
    first_host: str | None
    last_host: str | None
    usable: int

def calculate(cidr: str) -> SubnetResult:
    net = ipaddress.ip_network(cidr.strip(), strict=False)
    hosts = list(net.hosts())
    return SubnetResult(
        network=str(net.network_address),
        broadcast=str(net.broadcast_address),
        netmask=str(net.netmask),
        wildcard=str(net.hostmask),
        prefixlen=net.prefixlen,
        host_count=net.num_addresses,
        first_host=str(hosts[0]) if hosts else None,
        last_host=str(hosts[-1]) if hosts else None,
        usable=max(net.num_addresses - 2, 0),
    )
```

### 6.2 网络发现 `core/discovery.py`
- 多线程 ping 扫描目标网段，解析 IP / 主机名 / MAC（ARP 表或系统命令）。
- 通过 `AsyncWorker` 后台执行，`EventBus` 回传进度与单条结果，实时入 `DeviceStore`。
- 函数式接口：`scan_network(cidr: str, on_progress, on_host) -> list[Host]`。

### 6.3 连通诊断 `core/diagnostics.py`
- `ping(target, count)`：按 `platform.system()` 分支 `-n` / `-c`。
- `traceroute(target)`：分支 `tracert` / `traceroute`。
- `port_scan(target, ports)`：并发 `socket` 探测 TCP 端口。
- `dns_query(name, rdtype)` / `http_probe(url)`：解析与可达性探测。
- 全部返回结构化结果（`dataclass`），由 GUI 渲染。

### 6.4 性能测速 `core/speedtest.py`
- `Iperf3Server` / `Iperf3Client`：封装 iperf3 服务端/客户端（TCP/UDP，双向）。
- `ExternalTester`：外网测速（对接 speedtest 协议或自建 HTTP 探针），输出下载/上传/延迟/抖动/丢包。
- iperf3 二进制按平台从 `resources/` 加载，启动时检测缺失并提示。

### 6.5 仪表盘 `gui/dashboard.py`
- 订阅 EventBus 中 发现/测速/监控 事件，聚合渲染 KPI 卡（在线设备、平均延迟、吞吐、告警）、24h 流量折线、设备类型分布、近期设备列表。

### 6.6 编解码工具 `core/codec.py`
- `base64_encode/decode`、`url_encode/decode`、`hex/bin/dec/oct` 互转。
- `hash_data`（MD5/SHA1/SHA256/SHA512/CRC32）、`jwt_parse`、`timestamp_convert`、字符串处理、PEM/证书解析。
- 纯计算，无外部依赖，桌面端优先落地。

## 7. 基础设施设计

- **TabRegistry**：`register(TabDescriptor)`，`TabDescriptor{id, title, icon, factory, stage}`；按 `tab_id` 懒加载构建/切换，避免冷启动卡顿。
- **AsyncWorker**：`QRunnable` + `threading.Event` 协作停止；信号 `started / progress / finished / error`；任务取消只在检查点响应，禁用强杀。
- **EventBus**：轻量发布/订阅（`subscribe(topic, cb)` / `publish(topic, payload)`），解耦模块与仪表盘、告警中心。
- **Container**：服务注册与注入（core 服务以单例注册，GUI 经构造注入）。
- **Theme**：light/dark 设计 token + QSS 渲染；跟随系统或手动切换。
- **i18n**：zh_CN / en_US 资源（JSON），`tr(key)` 取词。
- **Crypto**：凭据 AES 加密存储（`data/.crypto_key`），不落明文。

---

## 8. 数据模型与存储

- **Config**：`data/config.yaml` 应用配置（主题、语言、默认网段、iperf3 路径）。
- **DeviceStore**：`data/devices.json` 发现设备库（IP、MAC、厂商 OUI、主机名、位置、类型）。
- **History**：测速/扫描历史，供趋势分析（`data/history/`）。
- **凭据**：经 `Crypto` 加密落盘，密钥存 `data/.crypto_key`（首次启动随机生成）。
- **IPAM**：网段/分配记录（`data/ipam.json`），与发现结果自动对账。

---

## 9. 编码规范

- 全量类型注解；`core/` 禁止 import PySide6；模块 docstring 齐全；函数单一职责。
- 命名：`core` 用 snake_case 函数/类；GUI 用 Qt 风格（子类 `QWidget`）。
- 提交前 `pytest` 通过；新功能配套单测（至少覆盖 core 纯逻辑）。
- 跨平台命令统一走适配层（`platform.system()` 分支），不得硬编码 Windows 路径。

## 10. 开发路线

- **阶段 0 · 工程骨架**：`pyproject.toml` + 分层目录 + 应用外壳 + 基础设施（TabRegistry / 主题 / i18n / 加密 / 异步 / 事件总线 / DI）+ 空仪表盘壳 + 左侧导航（18 模块占位，按阶段点亮）。
- **阶段 1 · MVP（可发布）**：子网计算器 → 网络发现 → 连通诊断 → 性能测速 → 仪表盘；附编解码工具。
- **阶段 2 · 能力补全**：配置管理、监控告警、IPAM、专项排障、安全管理、抓包、流量深度分析、带外与机房、音视频物联、工具箱补全、报表自动化。
- **阶段 3 · 高阶**：AI 助手、集成 API、平台权限（RBAC）、插件市场。

**建议启动顺序**：骨架 → 子网计算器（验证架构）→ 发现 → 诊断 → 测速 → 仪表盘 → 编解码。

## 13. 风险与对策

| 风险 | 对策 |
|------|------|
| 跨平台命令差异 | 统一适配层（`platform.system()` 分支） |
| iperf3 二进制缺失 | 启动时检测并提示下载/内置 |
| 后台任务卡死 | 协作式取消，禁用线程强杀 |
| 凭据安全 | 全程 AES 加密，不落明文 |
| 大网段扫描慢 | 并发上限 + 超时控制 + 进度回传 |
| 特权操作受限（SNMP/SSH/抓包） | 受 RBAC 管控，明确权限提示 |

## 14. Web 可行性备忘（暂不做）

18 模块理论上均可 Web 化（配后端代理执行真实操作）。其中编解码/子网/工具箱计算类可纯前端；其余需 FastAPI 后端复用同一 `core` 引擎。因 `core` 与 UI 解耦，未来接 Web 几乎零成本——**当前阶段不实现**。
